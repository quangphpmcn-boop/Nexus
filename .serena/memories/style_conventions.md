# Style & Conventions

## Language
- User-facing content: **Vietnamese**
- Code, commits, variable names: **English**
- Forward slashes in paths: `C:/Users/...` not `C:\Users\...`

## Commits
- Atomic commits, conventional format
- Include docs when relevant (`commit_docs: true`)

## File Guidelines
- Max 500 lines per file (`performance.max_file_lines`)
- Warn on file creation (`warn_on_create: true`)

## Naming
- Workflow files: kebab-case (`session-handover.md`)
- Skill folders: lowercase (`frontend`, `backend`)
- Agent files: kebab-case with role prefix
